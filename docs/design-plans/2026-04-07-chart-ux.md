# Chart UX Improvements Design

## Summary

The chart UX improvements address three independent defects in the stock analyzer's chart interface. A CSS regression introduced in commit `bdab7e3` — which switched the HTML entrypoint from the compiled Tailwind stylesheet to a base stylesheet — silently dropped all component-specific styles, breaking the measurement overlay and the right-click drag-zoom indicator. The fix reconciles the two stylesheets so the full compiled CSS is served again. Separately, a tooltip on the Plotly chart toolbar is mispositioned and blocks adjacent toolbar buttons; a targeted CSS override corrects its placement.

The third and most significant change is a backend performance fix for historical data loading. When zooming out to multi-year time ranges, the data service currently discards a full database of cached historical prices if the most recent entry is more than seven days old, forcing a slow full API fetch (30-60 seconds). The fix replaces this all-or-nothing staleness check with a targeted gap-fill strategy: the bulk of historical data is read from the database (~10ms for 20 years), only the missing recent days are fetched from the API (a small, fast request), and the gap-filled prices are persisted so subsequent requests are fully cached.

## Definition of Done

1. **CSS regression fix:** Restore all styling lost in commit `bdab7e3` (switched from `styles.css` to `base.css`), including measurement overlay, drag-zoom region, and any other affected components. All previously-working UI features should work again.

2. **Zoom-out performance:** Zooming out to longer time ranges (5yr, 10yr, max) loads historical data in under ~5 seconds instead of 30-60 seconds. The chart remains responsive during loading.

3. **Toolbar tooltip fix:** The "Zoom" tooltip on the Plotly modebar is mispositioned and blocks clicking adjacent icons. Fix tooltip positioning so all toolbar icons are clickable.

4. **No regressions:** Existing chart features (measurement tool, right-click zoom, scroll zoom, multi-series mode, technical indicators, range extension on zoom past data bounds) all continue working after fixes.

## Glossary

- **Tailwind CSS**: Utility-first CSS framework. The project compiles `src/input.css` through Tailwind to produce `styles.css`; `base.css` bypasses this compilation and lacks component-specific styles.
- **Measurement overlay**: Custom chart interaction showing a shaded region, vertical boundary lines, and a floating bubble with percentage return and dollar change when click-dragging across the chart.
- **Drag-zoom region**: Amber-highlighted bounding box drawn during right-click drag to define a zoom area on the chart.
- **Modebar**: Plotly's built-in floating toolbar above the chart with icons for camera (screenshot), magnifier (zoom), and other interactions.
- **`AggregatedStockDataService`**: Backend service responsible for fetching, caching, and serving historical stock price data, sitting between API controllers and database/external API providers.
- **Sparsity check**: Guard that rejects database results covering fewer than 20% of expected trading days, indicating the database genuinely lacks data for a symbol.
- **Gap detection / gap-fill**: New logic that identifies how many recent trading days are missing from database results and fetches only those days from the external API.
- **Request coalescing**: Pattern that deduplicates concurrent in-flight requests for the same data, so two simultaneous zooms don't trigger two API calls.
- **`PriceRefreshService`**: Daily crawler service that populates historical prices in the database.
- **BB, SMA, RSI, MACD, Stochastic**: Technical analysis indicators (Bollinger Bands, Simple Moving Average, Relative Strength Index, Moving Average Convergence Divergence, Stochastic Oscillator).
- **Multi-series mode**: Chart mode rendering multiple stock symbols for comparison or benchmark overlay.

## Architecture

Three changes: CSS regression fix (frontend), performance fix (backend), and tooltip fix (frontend).

### CSS Regression Fix

Commit `bdab7e3` switched `index.html` from `css/styles.css` (compiled from `src/input.css`) to `css/base.css`, dropping all styles defined in `src/input.css`. This broke at minimum the measurement overlay (and likely other components Patrick noticed).

The JS code in `dragMeasure.js` is fully functional — it creates all DOM elements dynamically via `_createOverlayDOM()`. The CSS in `src/input.css` (lines ~1262-1396) defines styling for `#drag-measure-overlay`, `#drag-measure-region`, `#drag-measure-bubble`, `#drag-measure-start-line`, `#drag-measure-end-line`, and `#drag-zoom-region`.

The fix ensures all styles from `src/input.css` are included in whatever CSS file `index.html` references. This requires investigating what `base.css` contains vs what `styles.css` contained, and reconciling the two — either by reverting to `styles.css`, merging the missing styles into `base.css`, or rebuilding the Tailwind compilation pipeline.

### Hybrid DB + API Gap-Fill (Performance)

The staleness check in `AggregatedStockDataService.TryGetFromDatabaseAsync` currently rejects the entire DB result if the latest price is >7 days old — even with 20 years of good historical data. This forces a full API fallback (30-60s) when the crawler hasn't run recently.

**New fetch flow in `FetchHistoricalDataCoreAsync`:**

1. Query DB for the full requested date range (fast, ~10ms for 20 years).
2. **Sparsity check** (unchanged): If DB has <20% of expected trading days, fall through to full API fetch.
3. **Gap detection** (new): If latest DB price is older than the previous trading day, calculate the gap.
4. **Gap-fill from API**: Fetch only the missing recent days from the API provider (small request, fast). Merge with DB data.
5. **Persist gap-fill**: Store the new prices in the DB so subsequent requests are fully cached.
6. Cache the merged result for 1 hour (unchanged).

The staleness check is replaced by gap detection + targeted fill. The sparsity check remains as a guard against genuinely empty data.

### Modebar Tooltip Fix

Plotly renders modebar tooltips via `data-title` attributes using CSS `::before`/`::after` pseudo-elements. The "Zoom" tooltip currently overlaps the magnifier icon, blocking clicks.

CSS override in `input.css` repositions all modebar tooltips below the icons with sufficient offset. Applied to all modebar buttons for consistency.

## Existing Patterns

### Data Service Caching
`AggregatedStockDataService` uses `MemoryCache` with 1-hour TTL and `ConcurrentDictionary`-based request coalescing. The hybrid fetch follows the same caching pattern — merged results are cached under the same key format (`history:{symbol}:{fromDate}:{toDate}`).

### DB Persistence
Gap-filled prices are persisted using the existing `IPriceRepository` pattern (`SqlPriceRepository.AddPricesAsync` or equivalent). This follows the same flow as the daily crawler (`PriceRefreshService`).

### CSS Architecture
The measurement CSS in `src/input.css` follows the existing pattern of component-scoped selectors with CSS variable theming (`--measure-bg`, `--measure-line`, `--zoom-bg`, `--zoom-border`) and dark mode overrides via `.dark` selector.
