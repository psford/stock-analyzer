# Human Test Plan: Chart UX Improvements

**Implementation Plan:** `docs/implementation-plans/2026-04-07-chart-ux/`
**Date:** 2026-04-08

## Prerequisites
- Stock-analyzer running locally (`dotnet run` from `src/StockAnalyzer.Api/`)
- Database populated with historical price data (at least one symbol with 5+ years of data)
- All automated tests passing: `dotnet test tests/StockAnalyzer.Core.Tests/ --filter "AggregatedStockDataServiceTests"`

## Phase 1: Measurement Overlay and Visual Styling

| Step | Action | Expected |
|------|--------|----------|
| 1 | Navigate to the app in Chrome. Open a stock chart (e.g., AAPL). | Chart renders with price data and modebar icons visible. |
| 2 | Left-click and drag horizontally across a section of the chart. | A blue shaded region appears between the drag start and current cursor position. A solid vertical line marks the start point; a dashed vertical line follows the cursor. |
| 3 | Release the mouse button after dragging. | A floating bubble appears showing: percentage return (e.g., "+5.2%") and dollar change (e.g., "+$8.50"). The bubble should have a backdrop blur effect. |
| 4 | Right-click and drag horizontally across a section of the chart. | An amber/yellow-highlighted bounding box appears during the drag, indicating the zoom selection region. |
| 5 | Release the right-click drag. | The chart zooms into the selected region. The amber bounding box disappears. |
| 6 | Click the dark mode toggle (if available) or apply dark theme via browser/app settings. | App switches to dark theme. |
| 7 | Repeat steps 2-5 in dark mode. | Measurement bubble has darker background and lighter borders. Overlay elements (blue region, vertical lines, amber box) are clearly visible against the dark chart background. |
| 8 | While in dark mode, navigate to the dashboard view. | Tile cards render with proper dark-theme styling (no white backgrounds, readable text). |
| 9 | Check the watchlist. | Star indicators appear and are visible in dark mode. |
| 10 | Check price changes on any stock. | Up/down color coding (green/red) is correct and visible in both themes. |
| 11 | Check tab navigation. | Active tab state highlights correctly in both themes. |
| 12 | Open a Flatpickr date picker (if present on the page). | Date picker is themed to match current light/dark mode. |

## Phase 2: Data Loading Performance (Wall-Clock)

| Step | Action | Expected |
|------|--------|----------|
| 1 | Select a symbol with 10+ years of data (e.g., AAPL, MSFT). Click "Max" range button. Start a stopwatch when clicking. | Chart loads and renders all historical data in under 5 seconds. |
| 2 | Ensure the data crawler has not run for 7+ days (or manually set the latest price date back 7 days in the DB). Select the same symbol. Click "Max" range. | Chart loads in under 5 seconds. Data should extend up to today (gap-fill occurred). Verify the most recent data point is from today or the last trading day. |

## Phase 3: Modebar Tooltips and Interactions

| Step | Action | Expected |
|------|--------|----------|
| 1 | Hover over each modebar icon (camera, zoom in, zoom out, autoscale, reset axes) one at a time. | Tooltip text appears below (not above or overlapping) each icon. Positioning is consistent across all icons. |
| 2 | While a tooltip is visible on one icon, move the mouse to click an adjacent icon. | The adjacent icon responds to the click. No tooltip interference blocks the click target. Specifically test the zoom icon (originally reported broken). |
| 3 | Click each modebar icon in sequence: camera (download), zoom in, zoom out, autoscale, reset axes. | Each icon performs its expected action without errors. |

## Phase 4: Multi-Series and Indicators

| Step | Action | Expected |
|------|--------|----------|
| 1 | Open a stock chart (e.g., AAPL). Add a comparison overlay (e.g., SPY). | Both series render with distinct colors. Legend entries show both symbols. Both have working hover tooltips showing respective values. |
| 2 | Enable Bollinger Bands (BB) indicator. | Upper, middle, and lower bands render around the price series. |
| 3 | Disable BB. Enable SMA. | Simple Moving Average line renders as a smooth overlay on the price chart. |
| 4 | Disable SMA. Enable RSI. | RSI renders in a subplot below the main chart with values between 0-100. |
| 5 | Disable RSI. Enable MACD. | MACD renders in a subplot with signal line, MACD line, and histogram bars. |
| 6 | Disable MACD. Enable Stochastic. | Stochastic oscillator renders in a subplot. |
| 7 | Toggle each indicator on and off rapidly. | No rendering errors, no console errors, chart remains stable. |
| 8 | Scroll the mouse wheel over the chart. | Chart zooms in/out smoothly centered on the cursor position. |
| 9 | Double-click on the chart. | Chart resets to the default zoom/view level. |

## End-to-End: Full User Journey

1. Open the app fresh. Navigate to AAPL chart.
2. Click "Max" range — confirm data loads in under 5 seconds.
3. Use the measurement tool (left-click drag) to measure a 1-year period — confirm shaded region, vertical lines, and percentage bubble appear.
4. Right-click drag to zoom into a 3-month window — confirm amber bounding box and zoom behavior.
5. Use modebar "Reset axes" to return to full view — confirm icon is clickable.
6. Add SPY as a comparison overlay — confirm dual series with distinct colors.
7. Enable RSI indicator — confirm subplot appears below.
8. Scroll wheel to zoom in on a volatile period — confirm smooth zoom with both series and indicator.
9. Double-click to reset view.
10. Toggle dark mode — confirm all overlays, indicators, and modebar tooltips remain functional and visible.
11. Toggle back to light mode — confirm no visual artifacts remain.
